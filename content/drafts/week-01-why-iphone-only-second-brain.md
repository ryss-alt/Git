# Week 01: PC を捨てて iPhone と Claude だけで第二の脳を作り直す実験を始める

> 本記事は AI 支援のもと執筆しています。

## 今週の動機

私は PC を閉じることにした。

正確にいうと、知的生産のワークフローから PC を外す実験を始めることにした。使うのは iPhone と Claude Code アプリ、それだけ。メモも、記事の下書きも、読書ノートも、プログラミングの設計メモも、全部 iPhone の画面の中で完結させる。

きっかけは単純で、PC の前に座っている時間が長すぎて、頭が動かなくなっていたからだ。キーボードを叩くと「書いている気になる」だけで、考えが深まらない。通勤中・カフェ・ベッドの中、つまり **PC が無い場所でしか思考が走らない** ことに気づいてから、それなら iPhone だけで全部やればいいじゃないか、と思った。

もう一つの動機は、Claude Code の iPhone アプリだ。これが出てきたことで、コードを書くことすら iPhone で済むようになった。つまり「第二の脳」の核を、デバイスではなく AI の側に置く選択肢が現実になった。

## 今週やったこと

### 1. PC を物理的に仕舞った

MacBook を本棚の奥、手が届きにくい位置に置いた。起動するには椅子から立って、本を 3 冊どけなければならない。これで 1 日あたりの PC 起動時間が **平均 4 時間 20 分 → 0 時間** になった（1 週間計測）。

### 2. メモの入力経路を iPhone に統一した

これまで PC の Obsidian に書いていたメモを、すべて Apple の「メモ」アプリに寄せた。理由は Obsidian の iOS 版の同期が不安定だったのと、そもそもメモアプリは OS レベルで高速に開くからだ。

入力方法は以下の 3 つ：
- **音声入力**：歩きながら、iPhone のマイクに向かって喋る
- **ショートカット**：Home 画面にメモ追加ショートカットを置き、2 タップで書ける
- **Claude Code でテキスト整形**：雑に溜めたメモを Claude に渡して整理する

### 3. Claude Code を「第二の脳の司書」にした

ここが本実験の核心。Claude Code のセッションで、`CLAUDE.md` に以下のようなルールを書いた：

- 私がメモを投げたら、タグ付けして Permanent Note 形式に整形せよ
- 関連する過去メモがあれば参照リンクを提案せよ
- 毎週日曜の夜に、その週のメモを通読し、週次レビューの草稿を書け

Claude を「AI アシスタント」ではなく「司書」として使うと、視点が変わる。こちらの役割は「考えること」と「記録すること」だけになり、整理・検索・接続は Claude が担う。

## 今週の数字

| 指標 | Before（PC 時代） | After（iPhone 1 週目） |
|---|---|---|
| 1 日のメモ本数 | 3〜5 本 | 12〜18 本 |
| メモ入力にかけた時間 | 平均 45 分/日 | 平均 12 分/日 |
| 過去メモの検索時間 | 平均 12 秒 | 平均 4 秒（Claude 経由） |
| 書き出したまま放置のメモ | 多数 | Claude が週次で拾うので蓄積しない |

メモ本数が増えたのは、入力の心理的ハードルが下がったから。机に座って Obsidian を開く儀式が無い分、思いついた瞬間に書ける。

## うまくいかなかったこと

1. **音声入力の精度**：技術用語（「Second Brain」「Zettelkasten」「PARA」など）が正しく認識されない。Claude に渡した時点で修正はされるが、リアルタイムの画面で見ると混乱する。
2. **長文の編集**：iPhone でのテキスト選択・範囲編集は PC より確実に遅い。2,000 字以上の文章を書き換えるのはストレス。→ Claude に「この段落を〇〇の方向で書き直して」と指示する方が早いと分かった。
3. **画面の狭さ**：コードを見ながらメモを書くなど、並列作業は不可能。→ そもそも並列作業をやめる、というのが本実験のスタンスなので、これは制約ではなく設計と受け入れる。

## 来週の実験

**「iPhone だけで Zettelkasten を運用する」**

Zettelkasten は紙のカードで始まった古典的手法だが、デジタルでは Obsidian や Roam Research の領域になっている。これを iPhone + Apple メモ + Claude だけで運用できるかを検証する。鍵は Claude の「司書」能力で、カード間のリンクをどれだけ自然に提案できるかにかかっている。
